# Academic Paper Interpreter / 学术论文解读

## Skill Overview
This skill leverages **mineru-document-extractor** for high-fidelity academic layout parsing (including LaTeX equations and Markdown tables) and utilizes Claude to perform logical extraction and structured synthesis. The objective is to transform raw PDF data into four distinct, standardized Markdown reports.

---

## 1. Core Dependencies
* **Parser**: `mineru-document-extractor` skill.
* **Input**: Native Markdown output from MinerU (no additional pre-cleaning required).

---

## 2. Deep Parsing Pipeline

### 2.1 Extraction Strategy
Scan the raw Markdown generated by MinerU. Preserve all **LaTeX** formatting for chemical/physical formulas and keep **Markdown Table** structures intact for data fidelity.

### 2.2 Cognitive Layer: Research Logic Analysis
The system simulates the reading process of a researcher:
* **Skimming**: Extracting research anchors from the Abstract and Conclusion.
* **Workflow Deconstruction**: Identifying the "Research Pipeline" (e.g., Modeling $\rightarrow$ Parameterization $\rightarrow$ Simulation $\rightarrow$ Analysis).
* **Evidence Chain**: Locating quantitative evidence in the Results section (e.g., $R^2$, success rates, or transport coefficients).
* **Boundary Identification**: Specifically searching for "Limitations," "Assumptions," or "Future Work" to define the scope of the study.

---

## 3. Required Outputs (4 Markdown Files)

Upon processing a document, the following four files must be generated:

### `SUMMARY_CARD.md` (Paper Abstract Card)
* **Title**: Full title of the paper.
* **Core Contribution**: A 1-2 sentence description of the specific problem solved and the proposed solution.
* **Key Quantitative Data**: Major metrics or coefficients extracted from the text.
* **Limitations Overview**: Brief summary of the study's boundaries (e.g., specific phases, length scales, or salt concentrations).

### `SECTION_SUMMARIES.md` (Section-by-Section Summaries)
Generate detailed summaries for each major section of the paper (e.g. Abstract, Introduction, Methodology, Results, Discussion, Conclusion). For example:

* **Abstract Summary**: Concise overview of the paper's core problem, approach, and key findings.
* **Introduction Summary**: Background context, literature review highlights, and research gap identification.
* **Methodology Summary**: Step-by-step explanation of the research methods, tools, and technical approach.
* **Results Summary**: Key findings, quantitative data, and experimental outcomes presented in the paper.
* **Discussion Summary**: Interpretation of results, comparison with existing literature, and implications.
* **Conclusion Summary**: Final conclusions, limitations, and potential future research directions.

Each section summary includes:
* Main points and key takeaways
* Supporting evidence and data
* Citations or references when applicable
* Visual descriptions of figures/tables (if present in that section)

### `METHODOLOGY_CARD.md` (Methodology Card)
* **Process Mapping**: A step-by-step breakdown of the research methodology.
* **Technical Stack**: Key tools, force fields, software (e.g., GROMACS/LAMMPS), or algorithms used.
* **Mathematical Definitions**: Extraction of core physical/chemical formulas in LaTeX format (e.g., Einstein/Green-Kubo relations).

### `REVIEW_CARD.md` (Critical Review Card)
* **Novelty**: Distinct improvements compared to existing SOTA (State-of-the-Art) methods.
* **Data Integrity**: Assessment of whether the results (Figures/Tables) directly support the conclusions.
* **Domain Impact**: The potential influence of this work on its specific sub-field.

### `CONCLUSION_CARD.md` (Conclusion Card)
* **Final Findings**: The ultimate physical laws verified or engineering goals achieved.
* **Actionable Insights**: Recommendations on whether to replicate the study or adopt the framework/logic for current projects.
* **Key Quotes**: 1-2 powerful excerpts from the original text that define the value of the research.

## 4. Execution Workflow
1.  **Parse Phase**: Execute `mineru-open-api` on the target PDF.
2.  **Read Phase**: Ingest the resulting `.md` file content and identify major sections (Abstract, Introduction, Methodology, Results, Discussion, Conclusion).
3.  **Synthesize Phase**: 
    * **Section Analysis**: Automatically detect and extract each major section from the parsed Markdown.
    * **Concurrent Generation**: Populate the five outputs (four synthesis cards + section summaries) based on the structured text:
      - Four synthesis cards (SUMMARY_CARD, METHODOLOGY_CARD, REVIEW_CARD, CONCLUSION_CARD)
      - Comprehensive section-by-section summaries (SECTION_SUMMARIES)
    * **Quality Control**: Strictly avoid hallucinating examples; all data must originate from the MinerU extraction.
    * **Formatting Preservation**: Maintain LaTeX syntax using `$ ... $` or `$$...$$` and Markdown table structures.
4.  **Finalize Phase**: Save each output as an independent file in the workspace.

---

## 5. Section Summary Generation Rules

### Detection Strategy
* **Automatic Section Parsing**: Identify section headers using Markdown heading patterns (`#`, `##`).
* **Content Grouping**: Aggregate paragraphs, figures, and tables under their respective section headings.
* **Boundary Handling**: Handle non-standard section structures (e.g., sections combined or split differently).

### Summary Quality Standards
* **Conciseness**: Each section summary should be 150-300 words, capturing essential information.
* **Completeness**: Include all quantitative data, key methodological details, and important findings from the section.
* **Clarity**: Written in clear, accessible language suitable for quick reference.
* **Linkage**: Where applicable, cross-reference related information from other sections.

### Output Format (`SECTION_SUMMARIES.md`)
Present all section summaries in a single consolidated Markdown file with:
* Table of contents linking to each section summary
* Consistent formatting and structure across all sections
* Preservation of any critical figures or tables referenced
* Markdown formatting (bold, italics, lists) for enhanced readability
